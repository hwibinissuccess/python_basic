피클, 언피클, 직렬화, 역직렬화의 차이
1)피클 pickle:객체를 바이트 스트림으로 변환하는 과정 (파일저장,네트워크 통신등)
2)언피클 Un-pickle: 바이트 스트림에서 객체로 복원하는 과정
3)직렬화(Serialization)  : 데이터 구조나 객체를 바이트 스트림으로 변환하는 과정
4)역직렬화(De-serialization) : 바이트 스트림에서 데이터구조나 객체로 복원하는 과정

임시기억장소 => 버퍼(기본단위 : 1byte), 클립보드(여러 개의 버퍼를 가지고 비주얼에서 사용, 복사 붙여넣기, 1byte 이상), 명령 레지스터(명령을 임시기억장치), 누산기가 있음.

with 키워드
 - 자원관리 자동화 -> 파일, 네트워크 사용 후 반드시 닫아야 하는데, 자동으로 자원 해제를 해결해줌
 - 예외 처리 단순화 -> 구문을 사용하다가 예외 발생시 with문이 try를 안 해도 자동으로 예외처리 코드를 단순화 시켜 리소스 해제를 한다.
  -> try-except –finally 구문을 전체 사용하지 않아도 된다.
 - 가독성 향상
 - 컨텍스트 관리자 만들기 : userClass로 __enter__ / __exit__를 재정의 한 메소드를 함께 사용할 수 있다.
